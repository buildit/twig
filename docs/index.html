Design Choices

The "Twig" application is designed with function in the forefront. The main sections of the application support viewing, editing, and manipulating the presentation of the visualization, or twiglet. Vertical side panels enable the user to work with environment settings, simulation settings, and filter down to specific nodes.

The dark color palette is borrowed from the established look of the DesignIt public-facing site, injecting BuildIt's signature blue color to highlight key elements. The nuanced darkness of the control panels fade into the backdrop as the white presentation area takes the foreground. This creates contrast and separation, while ensuring that the visualization remains as the most dominating visual element on the screen.

While responsive design is typically paramount, Twig is intended for large screens. Therefore, a minimum screen width of 1200px is recommended.

Buttons

Primary Button

<button class="button btn-sm">Primary</button>

<button class="button btn-sm disabled">Disabled</button>

The primary button denotes the most-desired action in an ideal user flow. It is reserved for affirmative, constructive actions, like "Save" or "Add". In any grouping of buttons, there should only be a singular primary button.

Disabled buttons are used to reveal potential action once all requirements of user input are met.

Secondary Button

<button class="button btn-sm btn-secondary">Secondary</button>

<button class="button btn-sm btn-secondary disabled">Secondary</button>

The secondary button is used for actions that are subsidiary or alternative to the main action item. It is used to defer to less affirmative actions like "Cancel" or "Close". In any grouping of buttons, secondary buttons are used for every button except the singular primary button.

Disabled buttons are used to reveal potential action once all requirements of user input are met.

Visualizations

Visualizations are made up of user-assigned icons which represent people, places, or things - called "nodes". Available options include the packaged FontAwesome library. The user has the ability to icon select color and use various colors for an icon type. Connecting lines depict relationships of the items, which are controlled by user display settings.
